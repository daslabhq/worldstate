What you're looking at

Each card below is one defineAsset() declaration from the scene-state library. Every asset comes with a default view that renders state at five sizes (icon · small · medium · large · xlarge — modeled on Apple WidgetKit + Daslab's WidgetSize) and three formats (HTML · Markdown · Text).

Same definition. Different consumers. The Markdown version is typically 3–5× cheaper in tokens than dumping raw JSON. The icon version is ~95% cheaper.

Two layers: Canonical types (Email, Message, Contact, …) are abstract primitives anyone can implement. Vendor implementations (Gmail, Slack, …) declare which canonical they extend.

Canonical: EmailMessageContactEventTaskDocument Vendors: GmailSalesforceSlackGoogleSheetsGoogleCalendarAirtableJiraNotionStripeGitHub

Canonical types

Abstract primitives. Use them directly, or as targets for vendor extensions.
